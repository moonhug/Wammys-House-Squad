# Tag Style Guide

Tags are flexible by design, but a few rules keep the archive clean.

## Recommended style

- Use lowercase.
- Prefer kebab-case.
- Use singular forms when possible.
- Reuse existing tags before creating a near-duplicate.

## Good examples

- `reverse-engineering`
- `steam-sale`
- `co-op`
- `action-rpg`
- `boss-rush`

## Avoid when possible

- `ReverseEngineering`
- `Reverse Engineering`
- `steam sale`
- `steam-sales`
- `coop` when `co-op` is already used

## Practical rule

- If two tags mean the same thing, pick one and stick with it.
- If a tag feels too broad, add a second tag instead of inventing a vague synonym.
